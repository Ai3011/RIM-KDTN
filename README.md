## Назначение основных модулей

### `backend/rs485_backend.cpp`
- Реализует протокол обмена с датчиками по RS‑485.
- Вычисляет CRC‑16 Modbus.
- Формирует запросы и обрабатывает ответы для команд:
  - `0x00` – чтение информации об устройстве (серийный номер, сборка, версия, тип).
  - `0x7E` – чтение служебных параметров (напряжения, ток, температура).
  - `0x7F` – широковещательное чтение и запись адреса устройства.
- Экспортирует функции `read_device_info`, `read_service_parameters`, `read_device_address`, `write_device_address`.

### `backend/rs485_backend.py`
- Загружает скомпилированную DLL через `ctypes`.
- Предоставляет Python-класс `RS485Backend` с методами-обёртками.
- Содержит функции `parse_response`, `parse_service_parameters`, `parse_address_response` – парсят сырые байты в удобные словари.

### `database/db_manager.py`
- Управляет SQLite базой данных.
- Определяет путь к БД.
- Создаёт таблицы:
  - `test_results` – результаты тестов (серийный номер, результат, диапазоны напряжений, длительность, временные метки).
  - `measurements` – поселкундные замеры напряжений и тока.
  - `settings` – настройки датчиков (порт, скорость, длительность, ручной режим, адрес).
- Функции для сохранения/загрузки результатов и настроек.

### `gui/main_window.py`
- Главное окно приложения.
- Инициализирует БД, загружает настройки, создаёт бекенд.
- Управляет 4 виджетами датчиков.
- Обеспечивает периодический опрос устройств (каждые 10 секунд).
- Запускает/останавливает тесты, обрабатывает кнопки «Запустить все», «Остановить все».

### `gui/sensor_widget.py`
- Виджет одного датчика – отображает:
  - серийный номер, сборку, версию, тип, температуру;
  - порт, скорость, длительность, ручной режим;
  - текущие значения напряжения, тока, ионистора, аккумулятора;
  - диапазоны (начало → конец) для ионистора и аккумулятора;
  - таймер обратного отсчёта;
  - два графика (напряжение ионистора и аккумулятора) в реальном времени.
- Кнопки управления: «Начать», «Стоп», «Повтор».
- Сигналы для связи с главным окном.

### `gui/settings_dialog.py`
- Диалог настройки параметров для каждого датчика:
  - выбор COM-порта (с обновлением списка);
  - выбор скорости (бод);
  - длительность теста (минуты + секунды);
  - отображение серийного номера (только для чтения).

### `gui/database_viewer.py`
- Окно просмотра результатов тестов.
- Фильтры по серийному номеру, актуальности, результату (положительные/отрицательные).
- Отображение всех полей из таблицы `test_results` с цветовой индикацией:
  - зелёный – положительный;
  - красный – отрицательный;
  - жёлтый – прерван.


### `main.py`
- Точка входа.
- Создаёт экземпляр `QApplication` и главное окно `MainWindow`.

## Как запустить проект

### Из исходного кода (Python)

1. **Установка Python библиотек:**
   ```bash
   pip install pyqt5 pyserial matplotlib pyinstaller
2. **Установка компилятора GCC (MinGW‑w64) в среде MSYS2:**

    ```bash
    pacman -S mingw-w64-ucrt-x86_64-gcc
3. **Сборка DLL :**

    ```bash
    g++ -shared -o rs485_backend.dll rs485_backend.cpp -static-libgcc -static-libstdc++ -lws2_32 -DBUILD_DLL

4. **Сборка .exe файла:**

    ```bash
    pyinstaller --onefile --name "KDTN" --add-binary "rs485_backend.dll;." --hidden-import "matplotlib.backends.backend_qt5agg" --hidden-import "matplotlib.backends.backend_qt5" main.py
